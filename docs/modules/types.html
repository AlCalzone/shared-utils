NoInfer

NoInfer<T>: T & {[ K in keyof T]: T[K] }
  • Defined in src/types/index.ts:251

Marking a parameter of a generic function with NoInfer causes its type to be inferred from other arguments with the same type instead of creating a union. Example:

const foo = <T>(arg1: T, arg2: T) => arg2;
foo({a: 1}, {b: 2})
// gets inferred as {a: number, b?: undefined} | {a?: undefined, b: number}
const bar = <T>(arg1: T, arg2: NoInfer<T>) => arg2;
bar({a: 1}, {b: 2})
// is an error: "a" is missing in type {b: 2}
